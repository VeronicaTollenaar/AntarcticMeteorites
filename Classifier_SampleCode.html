Sample code for classifying Positive and Unlabelled observations¶

Naive Bayes with multidimensional Kernel Density Estimation (KDE) and dimensionality reduction using a Principal Component Analysis (PCA)¶

The classifying algorithm, written in python using the scikit-learn library, consists of three steps. (i) First, the input data is transformed and standardized so that the mean is zero and the variance equals one. Then, a principal component analysis is performed to reduce the dimensionality of the data.

# open labelled (lab), unlabelled (unlab) and negative (neg) data (in this example randomly

unlab_raw = pd.DataFrame(np.random.randn(1000, 6)+11, 
                        columns=['feature_1','feature_2','feature_3',
                                 'feature_4','feature_5','feature_6'])
neg_raw   = pd.DataFrame(np.random.randn(80, 6)+11.2, 
                        columns=['feature_1','feature_2','feature_3',
                                 'feature_4','feature_5','feature_6'])


# transform data (if needed)

# standardize data
scaler = preprocessing.StandardScaler().fit(pd.concat([lab_raw,unlab_raw]).values)
lab_st   = scaler.transform(lab_raw)
unlab_st = scaler.transform(unlab_raw)
neg_st   = scaler.transform(neg_raw)


# perform principal component analysis
pca = PCA().fit(np.concatenate([lab_st,unlab_st]))
lab_pcs   = pca.transform(lab_st)
unlab_pcs = pca.transform(unlab_st)
neg_pcs   = pca.transform(neg_st)


# split up in train, calibration and test (in this example arbitrary)
lab_train   = lab_pcs[:50]
pos_cal     = lab_pcs[50:80]
pos_test    = lab_pcs[80::]
unlab_train = unlab_pcs
neg_cal     = neg_pcs[0:40]
neg_test    = neg_pcs[40::]

# select the number of principal components (defined by the amount of training data (see 
# "Silverman, B.W.. Denity Estimation for Statistics and Data Analysis. (Chapman an Hall, 
# 1986)")
pcmax = 2 # number of principal components

(ii) The multidimensional densities of the first x principal components of the labelled and unlabelled observations are estimated through a KDE with a ten-fold cross-validation to obtain the bandwidth parameter. To reduce computation time, a random subset of the observations can be used.

(iii) The decision rule is applied to classify observations as positive or negative.
$p\left(y=1\right)=\ \frac{n_{labelled}\times\lambda}{n_{labelled}\times\lambda+\ n_{unlabelled}}$ and $ p\left(y=0\right)=\ \frac{n_{unlabelled}}{n_{labelled}\times\lambda+\ n_{unlabelled}}$ (Eq. 1)

$\mathrm{p}\left(\mathrm{y=1}\middle|\mathrm{x}\right)\mathrm{\approx\ }\frac{\mathrm{p(x|s=1)\times\ p} \left(\mathrm{y=1}\right)}{\mathrm{p}\left(\mathrm{x}\middle|\mathrm{s=1}\right)\mathrm{\times p} \left(\mathrm{y=1}\right)\mathrm{+p} \left(\mathrm{x} \middle|\mathrm{s=0}\right)\mathrm{\times p(y=0)} }$ (Eq. 2a)

$\mathrm{p}\left(\mathrm{y}=0\middle|\mathbf{x}\right)\approx\ \frac{\mathrm{p}\left(\mathbf{x}\middle|\mathrm{s}=0\right)\times \mathrm{p}\left(\mathrm{y}=0\right)}{\mathrm{p}\left(\mathbf{x}\middle|\mathrm{s}=1\right)\times \mathrm{p}\left(\mathrm{y}=1\right)+\mathrm{p}\left(\mathbf{x}\middle|\mathrm{s}=0\right)\times \mathrm{p}\left(\mathrm{y}=0\right)}$ (Eq. 2b)

# calculate ROC curve (consisting of TPrate and FPrate) to empirically select the value for
# the cost parameter labda

# define possible values of the cost parameter labda (Eq. 1)
labdas = np.linspace(1,1000,100)

# define empty arrays for the true positive (TPrate) and false positive rates (FPrate)
TPrate = np.zeros(len(labdas))
FPrate = np.zeros(len(labdas))

# define a function to calculate p(y=1|x) and p(y=0|x)
def probs(input_scores_lab,input_scores_unlab,py1,py0):
    py1gx = (input_scores_lab*py1)/(input_scores_lab*py1 + input_scores_unlab*py0)
    py0gx = (input_scores_unlab*py0)/(input_scores_lab*py1 + input_scores_unlab*py0)
    return py1gx, py0gx

# loop over all values of the cost parameter labda
for labda in range(len(labdas)):
    # calculate p(y=1) and p(y=0)
    py1 = len(lab_train)*labdas[labda]/(len(lab_train)*labdas[labda] + 
                                   len(unlab_train))
    py0 = len(unlab_train)/(len(lab_train)*labdas[labda] + 
                                   len(unlab_train))
    # calculate p(y=1|x) and p(y=0|x) for positive and negative calibration data
    py1gx_pos_cal, py0gx_pos_cal = probs(scores_pos_cal_lab,scores_pos_cal_unlab,py1,py0)
    py1gx_neg_cal, py0gx_neg_cal = probs(scores_neg_cal_lab,scores_neg_cal_unlab,py1,py0)
    # calculate the true positive (TPrate) and false positive rates (FPrate)
    TPrate[labda] = len(py1gx_pos_cal[(py1gx_pos_cal)>0.5])/len(py1gx_pos_cal)
    FPrate[labda] = len(py1gx_neg_cal[(py1gx_neg_cal)>0.5])/len(py1gx_neg_cal)

# plot ROC curve
plt.plot(FPrate,TPrate)
plt.xlabel('False Positive rate')
plt.ylabel('True Positive rate');

# select value of the cost parameter (corresponding to an operating point on the ROC curve),
# using the harmonic mean of the precision and sensitivity estimated by the calibration data

# transform true positive and false positive rates to number of true positives, true
# negatives, false positives, and false negatives
TP = TPrate*len(pos_cal)
TN = (1-FPrate)*len(neg_cal)
FP = FPrate*len(neg_cal)
FN = (1-TPrate)*len(pos_cal)

# calculate the precision and the sensitivity and the harmonic mean between these two (F1)
precision = TP/(TP+FP)
sensitivity = TP/(TP+FN)
F1 = 2*precision*sensitivity/(precision+sensitivity)
# plt.plot(labdas,F1) # visualize

# select value of the cost parameter labda corresponding to the first maximum value of F1
